# PropValuate AI — Project Files

This folder contains the website as separate files (not one merged HTML) plus
the Python script used to train the valuation model.

```
index.html          → page structure
style.css            → navy / gold / emerald / coral theme
app.js               → valuation logic, chatbot, PDF export
data.js              → trained model + market data (generated by train_model.py)
leaflet.js/.css       → map library
jspdf.umd.min.js      → PDF report library
train_model.py        → trains the XGBoost model, outputs model.json + meta.json
requirements.txt       → Python packages needed for train_model.py
```

## Run the website in VS Code

The site is plain HTML/CSS/JS — it doesn't need a build step, but opening
`index.html` directly with a `file://` URL will block some browser features
(fetching local scripts can be flaky), so serve it locally instead:

1. Open this folder in VS Code (`File → Open Folder…`).
2. Install the **Live Server** extension (by Ritwick Dey) from the Extensions
   panel if you don't have it.
3. Right-click `index.html` in the file explorer → **Open with Live Server**.
4. It opens at `http://127.0.0.1:5500/index.html` and auto-reloads on save.

No Live Server? You can also just run, from a terminal in this folder:
```bash
python3 -m http.server 8000
```
then visit `http://localhost:8000`.

## Run the Python model script in VS Code

`train_model.py` retrains the model from a housing CSV and regenerates the
data the site runs on.

1. Open a terminal in VS Code (`` Ctrl+` ``) in this folder.
2. Create and activate a virtual environment (recommended):
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate      # Windows: .venv\Scripts\activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. **Update the CSV path** at the top of `train_model.py` — it currently
   points at `/mnt/user-data/uploads/1783659962212_Housing_full_enriched_clean.csv`,
   which was this session's upload path. Change it to wherever your housing
   CSV actually lives, e.g.:
   ```python
   df = pd.read_csv('Housing_full_enriched_clean.csv')
   ```
5. Run it (▶ button in VS Code, or from the terminal):
   ```bash
   python train_model.py
   ```
   This prints the model's R² / MAE and feature importances, and writes
   `model.json` and `meta.json`.

Note: `data.js` (used by the site) is a separate, already-baked JSON export —
regenerating `model.json`/`meta.json` from `train_model.py` doesn't
automatically update `data.js`. Let me know if you'd like a small script
added that converts `model.json` + `meta.json` into the `data.js` format the
site expects, so the two stay in sync automatically.
